# PM2008 Tools Preview

用于修改 PM2008 键位映射及部分其他设置的辅助工具。

> 仅适用于截止 2026/7/5 前发布的版本。由于游戏本体及相关功能仍处于测试阶段，设置不保证在所有环境下有效，也不保证能够获得预期效果。

## 系统要求

* .NET Framework 4.7.2 或更高版本

## 使用方法

1. 运行 config.exe；

2. 打开“其他设置”页面并选择游戏目录，选择后会立即保存，程序目录下会自动创建 config.json 配置文件。

工具负责生成完整的 launcher 启动命令。选择游戏目录后，工具会检查相关资源（launcher.exe、irom.bin、erom.bin、sram.bin）是否存在。启动时会自动读取配置并校正无效值，例如音频队列上限不能小于目标值、音量增益限制在 0–200%、显示倍率不能小于 1 等。未设置的模式会恢复为默认选项。

## 功能简介

### 键位映射

支持为 PM2008 的各项操作重新设置键位，并允许一个功能绑定两个按键及配置输入延迟。

工具会以挂起状态启动 launcher，等待 Windows Loader 完成导入表解析后，在目标进程中安装 GetRawInputData 的 IAT Hook。当 launcher 读取键盘 Raw Input 时，Hook 会先调用原始函数取得真实输入，再直接修改 RAWKEYBOARD 中的虚拟键码和扫描码，将设置的键位替换为游戏原生支持的目标键。例如将蓝鼓从 S 映射到 Q 后，按下 Q 时，游戏实际收到的是 S，并且默认的 S 则会被屏蔽，从而实现键位替换与交换。

Hook 代码由工具根据当前配置动态生成，并通过 VirtualAllocEx 和 WriteProcessMemory 写入目标进程，无需额外注入 DLL。按下与松开事件都会保持一致映射，输入延迟则通过延后 Raw Input 返回实现。该方案不影响其他程序，也不会修改 launcher 或其他游戏文件。

### 显示设置、仿真设置、音频设置

> 上述三种设置会在启动时转换为 launcher 命令行参数，不会修改游戏资源或系统显示配置。

显示设置主要用于修改普通窗口/无边框窗口模式，其他设置建议保持默认值。

仿真设置用于控制 launcher 内部的画面节拍和 A27 执行调度。除非排查速度、音画同步或特殊硬件兼容问题，否则建议保留默认值。

音频设置提供三种预设（低延迟、平衡和稳定），可依照自身电脑性能选择。
